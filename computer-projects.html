Computer Projects

Homelab, networking, security experiments, and the tools that keep everything running.

Overview

This section documents the computer-side of Delaplaine Digital Solutions: the Windows 11 Pro box that runs VMs, the NAS stack, routing and firewall projects, Pi-hole, and future SOC / malware testing environments. Over time each card below can become a full write-up with screenshots, configs, and code.

Related screenshots and diagrams can be stored under /assets/computers/.

Current & Planned Builds

VM-Based NAS & File Server

A virtualized NAS running on a Windows 11 Pro host, serving as centralized storage for photos, projects, and backups. Includes SMB shares, access control, and performance tuning work.

Ideas to cover later: disk layout, virtual disks, backup strategy, performance issues and fixes.

OPNsense Firewall & Network Segmentation

A dedicated OPNsense firewall VM / mini PC to separate lab traffic from household devices, experiment with rules, VLANs, and logging, and serve as the core of a more secure home network.

Potential content: WAN/LAN layout, DD-WRT as a switch, rulesets, logging, and lessons learned.

Pi-hole & DNS Filtering

A Pi-hole instance handling DNS for the network, blocking ads, trackers, and noisy domains. Ties into the firewall for better visibility and control of traffic.

Later: screenshots of the dashboard, blocklists used, and how it ties into the rest of the lab.

Homelab & Virtualization Stack

The main Windows 11 Pro machine runs multiple VMs: NAS, firewall, potential SOC tools, and sandboxes. This is the platform for testing, learning, and breaking things safely.

Good place to document: hypervisor choice, resource allocation, and what each VM is responsible for.

SOC Sim & Security Lab (Planned)

A simulated security operations center environment built from open source tools, log collection, and alerting pipelines to practice detection, investigation, and response workflows.

Future write-up could include: tool stack, sample attack scenarios, and how alerts are triaged.

Malware Testing / Sandbox Environment (Planned)

A controlled environment for analyzing suspicious files and behavior without risking daily-use machines. Will likely use isolated VMs, strict network rules, and snapshot-based recovery.

Later sections might show: network isolation diagrams, snapshot strategy, and any custom tooling or scripts.

Python & Utility Scripts

Small tools written in Python to scan the local network, gather system information, or automate repetitive tasks. These scripts tie together different parts of the lab.

Eventually: code snippets, GitHub links, and usage examples.
